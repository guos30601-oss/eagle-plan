# 雏鹰计划 PHP 后台部署与登录操作说明

这套后台用于正式上架后的账号发放、体验版控制、完整版解锁、学习进度记录和后台统计。

## 一、本机已经部署好的入口

本机使用 phpStudy 运行，当前端口为 `18080`。

学生登录入口：

```text
http://127.0.0.1:18080/login
```

后台管理入口：

```text
http://127.0.0.1:18080/admin/login
```

默认超管账号：

```text
账号：18888888888
密码：Eagle2026
```

第一次正式使用前，建议在后台新增自己的超管账号，再停用或修改默认账号。

## 二、phpStudy 需要启动什么

打开 phpStudy 后，确认下面服务为启动状态：

- Nginx
- MySQL 5.7
- PHP 7.3.4 NTS

本机数据库信息：

```text
数据库名：eagle_plan
账号：root
密码：root
```

如果以后换电脑部署，数据库账号密码不同，需要修改：

```text
eagle-backend/app/config.php
```

## 三、后台能做什么

后台主要做四件事：

- 新增学生账号
- 指定体验版或完整版
- 生成完整版解锁码
- 查看学生登录次数和学习进度

体验版开放：

```text
Day 1 - Day 3
```

完整版开放：

```text
Day 1 - Day 45
```

## 四、创建学生账号

进入后台后：

```text
后台首页 -> 学生账号 -> 新增学生
```

建议填写：

- 昵称：学生昵称
- 手机号：学生登录账号
- 密码：初始密码
- 解锁内容：体验版 / 完整版
- 开始日期：学生正式开始学习的日期

学生以后换手机、换电脑，只要用同一个手机号登录，学习进度会继续保留。

## 五、体验版升级完整版

当学生体验 3 天后购买完整版：

1. 后台进入 `解锁码`
2. 创建一个完整版解锁码
3. 把解锁码发给学生
4. 学生登录后输入解锁码
5. 账号升级为完整版

升级后不会重新开始，Day1-Day3 的记录会保留，学生从后面的进度继续学。

## 六、学习内容如何进入

学生不要再直接打开本地 HTML 文件。

正式使用时，统一从登录入口进入：

```text
http://127.0.0.1:18080/login
```

后台会根据账号权限判断哪些天可以打开。

学习内容文件在：

```text
eagle-backend/learning-content
```

后台会通过 `/learn/...` 统一访问这些页面。

## 七、正式上线时怎么做

正式服务器需要支持：

- PHP 7.3 或以上，推荐 PHP 8.1+
- MySQL 5.7 或 8.0
- Nginx 或 Apache

上线步骤：

1. 上传整个 `eagle-backend` 文件夹
2. 网站根目录指向 `eagle-backend/public`
3. 创建数据库 `eagle_plan`
4. 导入 `database/schema.sql`
5. 修改 `app/config.php` 中的数据库账号密码
6. 访问 `/admin/login` 登录后台

注意：网站根目录一定要指向 `public`，不要直接指向 `eagle-backend`。

## 八、常见问题

### 页面空白

通常是 PHP 没启动、数据库没连上，或网站根目录没有指向 `public`。

### 中文乱码

数据库字符集需要使用 `utf8mb4`。不要用记事本把项目文件另存为 ANSI。

### 体验版为什么 Day4 打不开

这是正常逻辑。体验版只开放 Day1-Day3，输入完整版解锁码后才能继续。

### 换设备后会不会重新学习

不会。进度跟手机号账号绑定，只要登录同一个账号，就会接着原来的进度走。
